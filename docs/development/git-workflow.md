# Git Workflow

## Branches

- main → Rama estable.
- feature/* → Desarrollo de nuevas funcionalidades.

## Commit Convention

- feat: Nueva funcionalidad.
- fix: Corrección de errores.
- docs: Documentación.
- refactor: Reorganización del código.
- test: Pruebas.
- chore: Infraestructura, configuración y mantenimiento.

## Examples

chore: initial project structure

feat: implement BIOS bootloader

feat: enter protected mode

fix: correct stack initialization

docs: update boot process

## Rules

- Un commit debe representar una única idea.
- No subir código que no compile.
- Documentar cambios importantes.
